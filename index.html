Registro Bagni – Guida all’installazione e configurazione

Questa guida spiega come installare l’app Registro Bagni, creare il database su Supabase, configurare le classi e importare gli studenti tramite CSV.

1️⃣ Creare un progetto su Supabase

  1. Vai su Supabase e crea un account (o fai login).

Se primo progetto vi sarà tutta la procedura di Sign Up da fare

  1. Clicca New Project.
  2. Inserisci:
  • Project name: il nome del tuo progetto (es. registro_bagni)
  • Password: scegli una password sicura per il database
  • Region: scegli la regione più vicina a te

  1. Attendi che Supabase crei il progetto (può richiedere qualche minuto).

2️⃣ Configurare il database con SQL

  1. Una volta creato il progetto, vai su SQL Editor nella dashboard di Supabase

.

  1. Copia e incolla lo script SQL qui fornito (non cambiare nulla!):

-- ================= ESTENSIONI =================

create extension if not exists "pgcrypto";

-- ================= STUDENTI =================

create table if not exists studenti (

  uuid uuid primary key default gen_random_uuid(),

  id text not null,

  id2 int8 not null,

  classe text not null,

  fuori bool default false,

  usciteoggi int8 default 0,

  superamenti int8 default 0,

  uscitats timestamptz,

  attivo bool default true

);

create unique index if not exists studenti_classe_id_unique

on studenti (classe, id);

-- ================= CLASSI =================

create table if not exists "Classi" (

  nome text primary key,

  attiva bool not null default true,

  "maxMaschi" int8 not null default 3,

  "maxFemmine" int8 not null default 3,

  "maxUscite" int8 not null default 2,

  "tempoMax" double precision not null default 5

);

-- ================= CONFIG (KEY-VALUE + ID LOGICO) =================

create table if not exists config (

  chiave text primary key,

  valore text

 

);

-- valori iniziali

insert into config (chiave, valore) values

('scuola', 'NOME SCUOLA'),

('adminPassword', '1234'),

('superAdminPwd', '123456')

on conflict (chiave) do nothing;

-- ================= AUTO INIT CLASSI =================

drop trigger if exists trg_auto_init_classe on studenti;

drop function if exists auto_init_classe();

create function auto_init_classe()

returns trigger as $$

begin

  insert into "Classi" (

    nome,

    attiva,

    "maxMaschi",

    "maxFemmine",

    "maxUscite",

    "tempoMax"

  )

  values (

    new.classe,

    true,

    3,

    3,

    2,

    5

  )

  on conflict (nome) do nothing;

  return new;

end;

$$ language plpgsql;

create trigger trg_auto_init_classe

after insert on studenti

for each row

execute function auto_init_classe();

CREATE TABLE uscite_giornaliere (

    id BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,

    classe TEXT NOT NULL,

    studente_id TEXT NOT NULL,

    ora_uscita TIMESTAMP NOT NULL,

    ora_rientro TIMESTAMP,

    minuti INTEGER,

    uscite_giornaliere INTEGER

);

  1. Esegui lo script selezionando tutto il testo appena incollato e cliccando su Run. Questo creerà tutte le tabelle necessarie.

3️⃣Cercare e copiare URL e ANON Key

  1. in alto a destra c’è un pulsante “Connect”, premere e cercare scheda “API Keys”; lì ci sono Project URL e ANON Key da incollare nella finestra di dialogo dell’app.

🔹 Puoi trovare questi dati anche nella dashboard di Supabase:

  • Vai su Settings → API
  • Copia il Project URL
  • Copia la anon public key
  1. Copia queste due stringhe ed inviale in posto facilmente raggiungibile da qualsiasi dispositivo (via maill)

4️⃣ Installare l’app

  1. Scarica e installa l’app Registro Bagni sul dispositivo o su desktop (Windows/macOS).
  2. Al primo avvio, ti verrà chiesto di inserire URL del progetto Supabase e ANON key.
  1. Inserisci questi dati nell’app.
  2. Accedi alla sezione Admin nell’app.
  3. Controlla che URL e ANON key siano salvati correttamente in memoria.

5️⃣ Importare gli studenti

  1. Prepara un file CSV (magari in excel e poi salvalo in csv) con ESATTAMENTE questa struttura come suggerito nel campo in cui incollare i dati:

nome,id,id2,classe

Mario Rossi,123,1,1A

Lucia Bianchi,124,2,1A

🔹 Ogni riga rappresenta uno studente:

  • nome: Nome e cognome
  • id: ID personale (stringa)
  • id2: sesso (int)    ->1=maschio, 2=femmina
  • classe: Classe di appartenenza
  1. Nell’Admin, clicca su Import CSV e incolla il contenuto del file.
  2. Attendere l’importazione (può richiedere qualche secondo/minuto).

6️⃣ Verifica studenti e classi

  1. Torna alla Home dell’app.
  2. Controlla che tutti gli studenti siano visibili.
  3. Puoi anche controllare direttamente su Supabase, nella tabella studenti.

✅ A questo punto l’app è pronta all’uso.
